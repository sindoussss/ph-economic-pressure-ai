# ph_economic_ai — In-App Honesty Cues (SP2a Design)

**Date:** 2026-06-11
**Status:** Approved (design), pending implementation plan
**Owner:** Sindous
**Program context:** First slice of SP2 (editorial polish), decomposed into SP2a (honesty cues — this) → SP2b (chart polish) → SP2c (real 3-sector charts) → SP2d (full editorial restyle). SP1 + SP3 are merged.

---

## 1. Problem & Goal

The app surfaces two very different kinds of number with the same visual weight: the **exploratory** swarm "agent agreement %" (an LLM-ensemble artifact that varies run-to-run) and the **validated** benchmark accuracy (reproducible, calibrated). A viewer can't tell which is which — and the agreement % reads like a confidence/probability it is not (the user hit exactly this: 82% one run, 62% the next).

**Goal:** make "exploratory vs validated" **unmistakable** in the UI, with one consistent visual language, so the app never overclaims — matching the honesty bar the whole project is held to. Wording/labels only; no new data, no layout overhaul.

---

## 2. Scope

### In scope (`ui/` labels + wording, offscreen-Qt tested)
- **Report — Swarm Consensus card:** an `· exploratory` tag by the heading; the agreement line reworded to *"N% agent agreement — varies per run, not a calibrated probability."*
- **Report — Validated-accuracy strip:** explicitly labelled **validated** (it already shows the efficiency verdict + the 90% calibrated interval), so it visually contrasts with the exploratory consensus.
- **Interact pane:** a one-line muted note that agent answers are exploratory.
- **Consistency:** reuse the existing "exploratory" wording already on the landing latest-forecast card and the sector-forecast card; converge on one small muted tag style/string so phrasing isn't scattered.

### Out of scope
- Charts (SP2b/c), the broad editorial restyle (SP2d), any change to how agreement is computed (we keep it stochastic — see the earlier decision), any benchmark/store change.

### Non-negotiable
- The validated-accuracy content stays factual and unchanged in substance; only its **label** ("validated") is added. The exploratory tags must not appear on the validated strip and vice-versa.

---

## 3. Components / changes

### 3.1 Shared honesty tag (`ui/honesty.py` — tiny helper)
A single source of truth so wording/style is consistent:
```python
EXPLORATORY = 'exploratory'
AGREEMENT_NOTE = 'varies per run · not a calibrated probability'
VALIDATED = 'validated'
def exploratory_tag() -> QLabel   # muted small pill, text 'exploratory'
def validated_tag() -> QLabel     # muted small pill, text 'validated'
```
(Or, if a QLabel factory is awkward to unit-test, expose the strings as constants + a `tag_style()` returning the stylesheet, and the panels build the QLabel — whichever keeps it testable. The strings are the tested seam.)

### 3.2 Report (`ui/stage4_report.py`)
- In the Swarm Consensus block (built in `populate_swarm`/`_build_swarm_left`): append the `exploratory` tag to the consensus heading, and reword the agreement line to include `AGREEMENT_NOTE`. Expose the rendered consensus text via a small `consensus_summary()`-style accessor (or assert against the QLabel text) for testing.
- In the Validated-accuracy strip: prepend/attach the `validated` tag.

### 3.3 Interact (`ui/stage5_interact.py`)
- Add a one-line muted caption under the Ask-an-Agent tab header: *"Answers are exploratory — generated by local agents, not validated."*

### 3.4 Landing / sector card (already say "exploratory")
- No behavioural change; if their wording differs from `EXPLORATORY`, align them to the shared constant (cosmetic consistency).

---

## 4. Testing
- `test_honesty.py`: the shared constants/helper exist; `exploratory_tag()`/`validated_tag()` (or the strings) carry the expected text.
- `test_stage4` (offscreen): after `populate_swarm(...)`, the report contains a label with `exploratory` near the consensus AND the agreement line contains "varies per run"; the validated-accuracy strip contains "validated".
- `test_stage5` (offscreen): the Interact Ask tab contains the "exploratory" caption.
- Full suite stays green.

## 5. Error handling
- Pure label additions; no new failure modes. Wrap any new label insertion in the existing try/except blocks that already guard `populate_swarm`'s sections so a label can't blank a card.

## 6. Deliverables (definition of done)
1. Shared honesty constants/helper (`ui/honesty.py`).
2. Report consensus tagged `exploratory` + reworded agreement line; validated strip tagged `validated`.
3. Interact exploratory caption.
4. Landing/sector wording aligned to the shared constant.
5. Tests per §4; full suite green.

## 7. Why it matters
It closes the one place the app could be accused of overclaiming — the agreement % that *looks* like a calibrated confidence but isn't. After this, every number on screen is visibly tagged as either exploratory (the swarm) or validated (the benchmark), so the app's honesty is legible at a glance — defense-relevant and consistent with the README's validated-vs-exploratory framing.
